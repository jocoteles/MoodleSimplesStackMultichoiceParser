Construa um script python que leia um arquivo yaml no formato stack_example.yaml e gere um arquivo xml no formato stack_example.xml.

O arquivo stack_example.xml é um arquivo atual exportado pelo Moodle. Todos as tags nele devem ser preservadas nessa mesma estrutura.

O arquivo stack_example.yaml é um arquivo que contém o formato em que as perguntas são construídas pelo usuário. A correspondência entre esse formato e o formato xml exigido pelo Stack Moodle é o seguinte:

- A chave 'category' no arquivo yaml corresponde à tag <category> no arquivo xml.
- A chave 'name' no arquivo yaml corresponde à tag <name> no arquivo xml.
- A chave 'penalty' no arquivo yaml corresponde à tag <penalty> no arquivo xml.
- A chave 'questiontext' no arquivo yaml corresponde à tag <questiontext> no arquivo xml.
- A chave 'questionvariables' no arquivo yaml corresponde à tag <questionvariables> no arquivo xml.

Todas as tags ausentes no arquivo yaml devem ser preservadas no arquivo xml, com seus respectivos conteúdos. Por exemplo, a tag <defaultgrade> não está presente no arquivo yaml, mas deve estar presente no arquivo xml.

Variáveis de output (que aparecem no texto da questão) no stack possuem o formato {@x@}, as quais devem corresponder a [[x]] no yaml.

Variáveis de input (que estão associadas às respostas da questão) no yaml possuem o formato {{x}}.

Tanto as variáveis de input, quanto de output, possuem modificadores opcionais separados por vírgula. Por exemplo, no yaml temos [[x, s:3, v:7]], onde 's' é o modificador de significantfigures (para input e output) e 'v' é o modificador de value (para output somente). Value é o valor da tag <value> no xml e significantfigures é a função que deve envolver a respectiva variável com o valor de algarismos significativos informados (vide stack_CAS.md e stack_example.xml). Os valores default de s e v são 3 e 0, respectivamente, em que s:0 implica que a função significantfigures não deve ser aplicada à variável. Se a questão yaml possuir a variável significantfigures declarada (por exemplo, significantfigures: 2), então ela é que define o s default que passa de 3 para 2 no exemplo.

A variável testoptions corresponde à tolerância percentual no valor a ser considerado correto. O valor default é 0.05.

A tag <questionnotes> no xml é fundamental, pois representa a identidade da questão quando os inputs são números aleatórios, permitindo o registro do histórico dos valores de fato fornecidos para o estudante. Ela não é explicitamente fornecida no yaml e deverá ser produzida a partir das variáveis de input da seguinte forma. Suponha que as variáveis de input sejam [[x]] e [[y]]. No xml deve ficar:
<questionnote>
    <text>x={@x@}, y={@y@}</text>
</questionnote>

A função rand nativa fornecida pelo Stack é bem específica e documentada em stack_CAS.md. Faça a seguinte conversão do yaml para o XML:

YAML -> XML
rand(a, b) -> a + rand(b.0)
rand(a) -> rand(a.0)
randint(a, b) -> a + rand(b)
randint(a) -> rand(a)
